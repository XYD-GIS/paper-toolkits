# 正文引用位置核对 · Inline Citation Check

> **何时用**：写完正文要交稿前，核对每个 [N] / (Author, Year) 都对得上参考文献列表，没有"幽灵引用"或"未被引用"
> **预期输出**：未引用文献清单 + 幽灵引用清单 + 缩写未解释清单 + 风格不一致清单

---

## 📋 完整 Prompt（复制下方代码块全部内容）

```text
# 角色
你是论文终稿校对的资深学术编辑。

# 任务
请核对我提供的【正文 + 参考文献列表】之间的引用一致性。

# 检查内容

## 1. 引用-文献对应（双向）
- **正文 → 列表**：正文中每个 `[N]` 或 `(Author, Year)` 是否在文献列表中能找到对应条目？
- **列表 → 正文**：文献列表中每条是否在正文出现过至少一次？
- 标记：
  - "幽灵引用"：正文引了但列表没有 → 必须补
  - "未引用文献"：列表有但正文没引 → 建议删（除非是 supplementary 中引的）

## 2. 引用风格统一性
- 全文是否一致用一种风格？
  - 上标编号：`[1]`、`[1,2]`、`[1-3]`（中文期刊、IEEE 常用）
  - 行内编号：`(Author, Year)`、`Author et al. (Year)`（APA、社科常用）
  - 数字括号：`[1]`（GB/T 7714、IEEE）
  - 圆括号编号：`(1)`、`[1]`
- **不能前面用上标后面用行内**

## 3. 引用顺序（GB/T 7714 / 中文期刊）
- 是否按正文出现顺序编号？
- 是否有"1 到 4，回到 5，又回到 1 到 4"这种乱跳？
- 如果文献编号需重排，给出新旧映射表

## 4. 缩写首次出现规范
扫描正文中**所有英文缩写**，检查每个缩写**首次出现**时是否给了中文/英文全称？
- 标准格式：`主成分分析（Principal Component Analysis, PCA）`
- 例外（行业通用，可不展开）：AI / GPU / CPU / DNA / RNA / LLM / DOI / ISBN

# 输出格式

## Part 1 [幽灵引用]
正文引了但参考文献列表里没有的：
- `[12]` 在正文第 X 段出现，但列表中无对应条目 → **必须补条目**
- ...

## Part 2 [未引用文献]
列表中有但正文未引的：
- 条目 #5（Zhang 2023）在正文中未引用 → **建议删除**
- ...

## Part 3 [风格不一致]
正文中混用的引用风格：
- 第 3 段用 `[1]`，第 7 段用 `(Zhang, 2023)` → **统一为一种**
- ...

## Part 4 [缩写未解释]
首次出现没给全称的英文缩写：
- 第 X 段："VIRB" → **未解释，需要给中文/英文全称**
- ...

## Part 5 [总结]
- 幽灵引用：N 条
- 未引用文献：N 条
- 风格不一致：N 处
- 缩写未解释：N 个
- 总体评价：[需要修订 / 基本可用 / 通过]

# 输入

## 正文
[在此处粘贴正文。可以是一节或整篇。]

## 参考文献列表
[在此处粘贴参考文献列表]
```

---

## 🛠 使用方法

1. 复制 prompt
2. 把正文 + 列表分别粘到对应占位符
3. 发送

⚠️ 输入很长（整篇论文 + 完整列表）时，AI 可能漏检。**建议分章节处理**。

## 💡 调优提示

- **正文超长**（> 10000 字）：分章节核对（每次输入 1-2 章 + 完整文献列表）
- **缩写多**：在 prompt 末尾加 "请重点检查 STEM 缩写，列出所有未首次解释的缩写"
- **只想检查某一项**：删除其他 Part 输出格式，只留你关心的

## 🔁 推荐自动化（LaTeX 用户）

如果你用 LaTeX 写作，**`pybtex` / `bibtex` + `chktex` 已自动检查"幽灵引用"和"未引用文献"**：

```bash
# 编译时会自动报警未引用 / 幽灵引用
pdflatex main.tex
bibtex main
pdflatex main.tex
pdflatex main.tex

# 看 .blg 文件，会列出 Warning--I didn't find a database entry for ...
```

或用 [latexdiff](https://www.ctan.org/pkg/latexdiff) 对比新旧版本，自动找出引用变化。
