# 引文位置与缩写规范（正文中）

---

# 一、引文位置规范

## 引文风格

- **上标编号**：`[1]` / `[1,2]` / `[1-3]`（中文期刊、IEEE 系常用）
- **行内编号**：`(Author, Year)` / `Author et al. (Year)`（APA、社科常用）

⚠️ **风格必须全文统一**——不能前面用上标后面用行内。

## 引文位置

| 情况 | 处理 |
|---|---|
| 通用观点 | 句末标注：`xxx 的研究表明 X 与 Y 显著相关[1].` |
| 具体方法 | 紧跟方法名：`本研究采用主成分分析（PCA）[2] 提取主要特征。` |
| 多文献并列 | 合并为区间：`已有研究[1-3] 探讨了 ……` |
| 同一文献多处引用 | 全文用同一个编号，不重复列入参考文献 |

## 引文顺序

GB/T 7714 与多数中文期刊：**按正文出现顺序编号**
- 正文第一次出现 [1]，第二次出现 [2]，依此类推
- 如果重新组织正文导致顺序变化，必须重新编号

参考文献编号必须按首次出现顺序严格递增，顺序乱跳属于硬伤。

# 二、缩写首次出现规范

**所有英文缩写首次出现时**必须给中文全称（或英文全称）+ 缩写。

| 错误 | 正确 |
|---|---|
| 我们采用 PCA 分析 | 我们采用主成分分析（Principal Component Analysis, PCA）|
| 在 RAG 任务上测试 LLM | 在检索增强生成（Retrieval-Augmented Generation, RAG）任务上测试大语言模型（Large Language Model, LLM）|

**例外**：行业通用缩写不必展开
- AI（人工智能）
- GPU、CPU
- JS、HTML、CSS
- DNA、RNA
- LLM（大语言模型）—— 顶会论文已普遍接受
- DOI、ISBN

⚠️ 不常见的英文缩写首次出现时必须附中文解释，否则读者无法识别其含义。
