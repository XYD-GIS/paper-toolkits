# 摘要写作骨架

## 标准 4 段骨架

```
段 1（背景+工作）：目标 ——【而】—— 样区/对象 占 X% —— 成为重要研究样区 —— 几项工作
段 2（方法）：基于 X 数据，采用 Y 方法，融合 Z 模型，开展什么研究
段 3（结果）：(1) 方法/数据优势 → (2) 时间/分类规律 → (3) 空间/对比规律 → (4) 影响因素 → 管理含义
段 4（结论）：最后一句落到能调什么或政策建议
```

经验法则：把目标和样区之间用"而"显式连接，让数据支撑样区的重要性。然后用"一是/二是"让读者数得出做了几件事。

---

## 段 1：背景 + 工作

反面 vs 正面对照：

| 反面（缺逻辑） | 正面（按骨架） |
|---|---|
| X 对领域 A 具有重要影响。Y 群体集中了……。探究驱动机制，是制定差异化策略的关键方法。 | 领域 A 已提出 [具体目标 + 时间节点]。**而** [对象/样区] 集中了全领域约 X% 的 [关键指标]，**成为** 该目标落实的**重要研究对象**。面向这一对象，本研究开展两项工作：**一是**构建 [时间跨度] 的 [数据/方法] 体系；**二是**识别 [自变量] 驱动 [因变量] 的非线性阈值与交互效应。|

### 段 1 写作检查

| 骨架环节 | 检查 |
|---|---|
| 目标具体化 | 是否写出具体年份/数字？不是抽象口号？ |
| "而"转折 | 目标句 → 样区句之间是否有**显式**转折？ |
| 数据支撑样区 | 是否给出占比数字？ |
| 样区定性 | 是否写了"成为……重要研究样区"？ |
| 工作数量外显 | 是否用"一是/二是"、"首先/其次"让读者**数得出**？ |
| 两项工作可区分 | 一是=数据/方法；二是=机制/分析。不要把两件事写成同一件。 |

---

## 段 3：结果骨架（4 条子条）

| # | 子条 | 必含要素 |
|---|---|---|
| ① | **数据/方法优势** | 与已有数据/方法比，精度好在哪（具体指标，如 r 系数、R²、行政尺度）|
| ② | **时间规律** | 增长最快的 X，最慢的 Y，拐点出现在 Z 年 |
| ③ | **空间/对比规律** | 最显著的 X；形态分类（带状/团状/中心-外围）；分组内部特征 |
| ④ | **影响因素** | 整体主导因子 A 和 B；不同分组主导因子；**最后一句落到"能调什么"** |

### 时间规律的写法陷阱

经验法则：你一句话里面想说五六个意思。到底想比增长速率、增长趋势，还是占比情况？

必须拆成独立断言，一句讲一个维度：
```
总量增长：X 由 a 增至 b
增速排名：增长最快的是 X，最慢的是 Y
拐点识别：A 拐点出现在 N 年；B 拐点出现在 M 年
占比演变：A 贡献增长总量的 64%
```

### 空间规律：必须用形状词分类

经验法则：直接看空间格局或空间集聚状——带状、亚铃状、多核心模式——这种命名才能让读者一眼看懂。

合法分类：带状 / 团状 / 中心-外围 / 哑铃型 / 多中心 / 单中心
不合法：双核驱动、港口集聚、X-Y-Z 走廊、三大核心区

### 影响因素：先整体，后分群，最后落管理含义

经验法则：整体上 A 和 B 是主导因子；而三个分组的分群分析各有不同；最后要点一下"能调什么"。

**管理含义落点模板**：
> 首要因子虽为 X，但 X 刚性较强、难以短期调整；真正可操作的抓手是 Y 指标，它们对应可执行措施。
