# 写作准则详解（11 条）

> 每条准则均附正反示例。输出前必须逐条自查，发现违反立即修正。

---

## 准则 1：中心句放段首

每段第一句话必须概括全段核心。评审委员只读首句就要知道这段讲什么。

经验法则：评审委员只读每段第一句话就能知道你做了什么，这一刻你的水平开始起跑。

| 反面 | 正面 |
|------|------|
| 张等发现 X。王等构建了 Y。李等基于 Z 模型…… | 该领域研究主要集中于 X 和 Y 两类。X 方面，可通过 ……；Y 方面，已有研究探明了 ……。|
| 2018 年指标为 XX，2019 年为 YY…… | 该指标持续上升，其中 A 组增幅最大（从 XX 增至 YY），B 组次之。 |

检查方法：遮住段落只看首句，能否知道全段在讲什么？不能 → 重写首句。

---

## 准则 2：标题即主语

标题叫什么，段落就以什么开头。主语绝不藏在句子中间。

经验法则：把主语藏在句子里面，读者理解不了。看了两页都没看懂在说什么——论文投出去被拒稿就是这种写法。

| 反面 | 正面 |
|------|------|
| 本研究在两个阶段使用 CatBoost 模型…… | CatBoost 模型主要应用于研究的两个阶段。 |
| 实验数据涵盖了多种来源…… | 实验数据包括 A 和 B 两部分，定义如下…… |
| 为了保证模型精度，本研究对比了六种算法…… | 模型选择实验采用了六种常见的机器学习算法，CatBoost 显著优于其余模型（表 X）。 |

检查方法：段落首句的语法主语是否与小节标题一致？不一致 → 调整。

---

## 准则 3：一个动词原则

一句话尽量只有一个动词加一个宾语。删掉所有冗余动词结构。

经验法则："进行""展开""开展""实现"这类壳子动词永远是多余的。"对 XX 进行了详细的调研"——直接写"调研了 XX"。

| 必须删除的结构 | 替换为 |
|---------------|--------|
| 对 XX 进行了分析 | 分析了 XX |
| 对 XX 进行了详细的调研 | 调研了 XX |
| 展开了 XX 的研究 | 研究了 XX |
| 开展了 XX 工作 | 完成了 XX |
| 实现了对 XX 的优化 | 优化了 XX |

检查方法：数每句话的动词数量。超过 2 个 → 必须精简。

## 准则 4：禁止模糊空话（零容忍）

经验法则："存在明显差异"——读者根本不知道你在想什么。"我有重要价值"——没用，要写出"能将准确率提升到 60%"才有信息量。

### 绝对禁用词表（出现即改，无例外）

| 禁用 | 必须替换为 |
|------|-----------|
| 存在明显差异 / 差异明显 / 显著分异 | 写出具体差异内容："呈现单中心型、带状型、多核心型三种聚集模式" |
| 呈现明显的梯度特征 / 梯度明显 | 写出具体梯度："核心区占 XX%，边缘区占 YY%，分支区占 ZZ%" |
| 至关重要 / 举足轻重 / 非常重要 | 改为具体能力描述："能够将 R² 提升 0.11" |
| 非常明显 / 十分显著 / 效果显著 | 改为具体数字："R² 从 0.689 提升至 0.799" |
| 重要意义 / 重要价值 / 重大贡献 | 改为具体解决了什么问题 |
| 具有一定的参考价值 | 改为具体为什么提供了什么参考 |
| 理想 / 良好 / 优异 | 改为具体指标："R² = 0.948, RMSE = XX" |
| 关键任务 / 关键方法 / 关键抓手 | 改为具体动作："构建 XX 数据集""识别非线性阈值" |
| 贡献总增量 / 贡献总增长 | 改为明确占比："占增长总量的 64%" |
| 趋于平缓 / 趋于稳定（单独出现） | 必须给出对比基准："A 组在 2020 年后增速放缓（相对 2015-2019）" |
| 已有研究表明 / 研究表明（句首） | 直接写结论，删除虚引导："X 与 Y 正相关" |
| 在 XX 方面（段首/句首） | 让 XX 作主语 |
| 差异化策略 / 差异化分析 | 改为具体差异内容 |

检查方法：全文搜索上述禁用词，逐个替换。一个都不能留。

## 准则 5：消灭"对"字

"对"不是真正的动词，只会让句子变模糊。

经验法则："对 X 的驱动机制"——你说能够改善什么、能够完善什么，做的事情就说得很清楚；用了"对"反而把动作藏起来。

| 反面 | 正面 |
|------|------|
| 对 X 的驱动机制 | 驱动 X 的机制 |
| 对模型精度进行验证 | 验证模型精度 |
| 对实验结果的特征进行分析 | 分析实验结果的特征 |
| 对结果的可靠性对比 | 验证结果可靠性 |

检查方法：全文搜索"对"，逐个判断能否删除或替换为具体动词。

## 准则 6：句间逻辑关系

段落内每句话之间必须有逻辑关系，不能是无关句子的堆砌。

经验法则：每一句话之间又没个逻辑关系，动不动什么张等什么、王等什么——这个东西有啥关系呢？

四种合法逻辑关系：
- **并列**："A 方面，……；B 方面，……"
- **递进**："首先获取 X，进而将其应用于 Y"
- **因果**："由于 A 精度最高，因此选择其作为最终模型"
- **转折**："已有研究集中于 A，但尚未涉及 B"

段落结构只有三种：总分、总分总、A 转折 B。

### "因此/所以"前后必须真因果

经验法则：你上面这么多废话，怎么能"因此"呢？哪能和这句话有关系？没有任何因果关系。

检查方法：
1. 全文搜索"因此 / 所以 / 故"
2. 遮住后一句，能否从前一句**推导**出后一句？
3. 推不出 → "因此"是连接剂不是真因果 → 删除或重写前句

示例：
| 反面 | 正面 |
|---|---|
| A 方法已发展多年，B 模型也被提出。**因此**获取最新 X 数据是精细化研究的基础。 | 主流方法有 A 和 B：A 的缺陷是 XX，B 的缺陷是 YY。**因此**需要构建更精细的核算数据集。 |

## 准则 7：总分结构

先告诉读者"一共几件事"，再逐一展开。

经验法则："A 包括 B、C、D；B 怎么样，C 怎么样，D 怎么样。"这就是一般化的逻辑、总分结构。每个正常的段落都是采用这种结构展开的。

| 反面 | 正面 |
|------|------|
| 首先做了 X，然后做了 Y，接着 Z，最后还加入 W…… | 数据预处理包括三个阶段：A、B、C。A 阶段，……；B 阶段，……；C 阶段，……|

### 工作数量必须外显

经验法则：该分一二的，你不分一二就是"首先写字再次"吗？你把它全写在一起，读者就难数出你做了几件事。

让读者"数得出你做了几件事"的三种工具：
1. **连词开头**：首先 / 其次 / 最后（三件及以下）
2. **编号**：一是 / 二是 / 三是；或 ①②③
3. **不同动词开头**：构建了 X；提出了 Y；优化了 Z

检查方法：读完一段，能否一口气说出"这段说了几件事"？说不出 → 工作数量未外显 → 加连词/编号/动词领起。

## 准则 8：字字有用

可有可无的字必然是无。

经验法则：如果这句话这个字在这篇论文当中可有可无，必然是无。

必须删除的虚词：对于、其中、始终、进而（无递进时）、从而（无因果时）。

### 虚引导词清单（必删）

| 虚引导 | 出现位置 | 处理 |
|---|---|---|
| 已有研究表明 / 研究表明 | 句首 | 直接写结论："X 与 Y 正相关" |
| 在 XX 方面 | 段首 / 句首 | 让 XX 作主语 |
| 将其作为 XX 进行 XX | 任意 | 拆成简单主谓宾 |
| 该方法已被应用于 | 任意 | "X 已应用于 A 和 B 两方面：……" |
| 基于 XX 方法（动宾前） | 动宾前 | 删除"基于"，直接用方法名作工具状语 |

## 准则 9：方法命名统一

经验法则：为什么有的方法用英文、有的用中文？有的方法标"中文（缩写）"，有的方法只写英文缩写？还有的方法只写中文？同一篇论文出现 3 种以上命名形态——就特别乱。

### 统一规则（二选一，不能混用）

**方案 A：全中文方法类别**

把英文缩写转译成中文方法类别。例如：
- PCA → 主成分分析
- t-SNE / UMAP → 降维可视化
- SHAP → 特征归因
- CatBoost / XGBoost → 梯度提升树

**方案 B：全"中文（英文缩写）"**

首次出现：**中文全称（英文缩写）** 或 **中文全称（英文全称，英文缩写）**
- 主成分分析（Principal Component Analysis, PCA）
- 偏依赖图（Partial Dependence Plot, PDP）
- 沙普利可加性解释（SHapley Additive exPlanations, SHAP）
- 卷积神经网络（Convolutional Neural Network, CNN）

### 人名方法的处理

Mann-Kendall、Sen、Moran、Dagum、CatBoost、SHAP、Adam 等是**人名或工具名**，不是缩写——保留英文原拼即可，不强制配中文译名。对应的**中文部分落在方法类型描述上**：
- Mann-Kendall **趋势检验**
- Adam **优化器**
- Dagum **基尼系数分解**
- **全局** Moran's I

检查方法：通读全文，方法命名风格是否一致。出现 3 种以上命名形态 → 违反。

## 准则 10：禁自造词 / 禁张冠李戴

### 类别命名必须用标准词

经验法则：一个要不就是带状，要不就是团状，要不就是中心-外围——这种形状。"港口集聚"又不是个形状。

合法的空间形态类别：**带状 / 团状 / 中心-外围 / 多中心 / 单中心 / 哑铃型 / 矩形型**

禁止自造：
- ❌ 双核驱动、港口集聚、三核轴向（不是形状）
- ❌ 自造缩写 + 错配"走廊"
- ❌ 把多个城市合称成一个"集聚区"

### 指标命名必须成立的中文搭配

经验法则：这两个词不能搭在一起，听起来就觉得是个病句。指标要叫得对。

❌ "处理量速度"、"性能效果"、"鲁棒鲁棒性"（语法搭配病句）
✅ "处理速度"、"处理通量"、"性能"、"效果"、"鲁棒性"

---

## 准则 11：改进要落到具体指标

经验法则：你不要在字上来回写半天，那好在哪我不懂。直接写"我把空间分辨率从 30 米提升到 10 米"——读者立刻懂。

### 禁止只说方法论标签

❌ 只说"本研究采用自下而上核算"——不够
✅ 必须紧跟具体指标提升：
- 空间分辨率：30m → 10m
- 行政单元：省级 → 市级 → 县级
- 核算门类：X 类 → Y 类
- R² / RMSE 的具体数字
- 覆盖期间：2015—2024（10 年面板）

### 批评要批到点子上

经验法则：表现也没表现到点上，批评也没说人家什么。

| 反面（空批评） | 正面（落到点） |
|---|---|
| 现有方法精度不足 | 现有方法只能达到 [粗粒度]，无法识别 [细粒度] 差异 |
| 现有指标存在局限 | 现有指标无法揭示 [关键维度] 的内部结构（如 A-B-C 三层关系）|
| 以往研究较为粗略 | 以往研究多使用单一时点/单一来源，缺乏 N 年/N 来源面板 |
