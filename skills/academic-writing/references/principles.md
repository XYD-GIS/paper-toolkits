# 写作准则详解（12 条）

> 每条准则均附正反示例。输出前必须逐条自查，发现违反立即修正。

---

## 准则 1：中心句放段首

每段第一句话必须概括全段核心。评审委员只读首句就要知道这段讲什么。

审稿人通常只读每段首句即判断段落内容。首句无法概括全段时，段落质量再高也难以传达。

| 反面 | 正面 |
|------|------|
| 张等发现 X。王等构建了 Y。李等基于 Z 模型…… | 该领域研究主要集中于 X 和 Y 两类。X 方面，可通过 ……；Y 方面，已有研究探明了 ……。|
| 2018 年指标为 XX，2019 年为 YY…… | 该指标持续上升，其中 A 组增幅最大（从 XX 增至 YY），B 组次之。 |

检查方法：遮住段落只看首句，能否知道全段在讲什么？不能 → 重写首句。

---

## 准则 2：标题即主语

标题叫什么，段落就以什么开头。主语绝不藏在句子中间。

主语藏在句子中间会直接降低可读性：读者需要多读数句才能确定段落对象，理解成本成倍增加。

| 反面 | 正面 |
|------|------|
| 本研究在两个阶段使用 CatBoost 模型…… | CatBoost 模型主要应用于研究的两个阶段。 |
| 实验数据涵盖了多种来源…… | 实验数据包括 A 和 B 两部分，定义如下…… |
| 为了保证模型精度，本研究对比了六种算法…… | 模型选择实验采用了六种常见的机器学习算法，CatBoost 显著优于其余模型（表 X）。 |

### 例证也要主语对齐

论述什么，就以什么作主语。前句的论断围绕类别，例证就以类别开头；前句的论断围绕任务，例证就以任务开头。例证开头与论断关键词错位，衔接即断。

| 反面 | 正面 |
|------|------|
| 已有研究多以单一数据集为评测对象，[方法 M] 的推理速度最快。 | 已有研究多以单一数据集或单一任务为评测对象：数据集上，[数据集 A] 使用最广；任务上，[任务 B] 涉及最多。 |

检查方法：段落首句的语法主语是否与小节标题一致？例句开头是否与所支撑论断的关键词一致？不一致 → 调整。

---

## 准则 3：一个动词原则

一句话尽量只有一个动词加一个宾语。删掉所有冗余动词结构。

"进行""展开""开展""实现"属于壳子动词，不承载信息。"对 XX 进行了详细的调研"应直接写为"调研了 XX"。

| 必须删除的结构 | 替换为 |
|---------------|--------|
| 对 XX 进行了分析 | 分析了 XX |
| 对 XX 进行了详细的调研 | 调研了 XX |
| 展开了 XX 的研究 | 研究了 XX |
| 开展了 XX 工作 | 完成了 XX |
| 实现了对 XX 的优化 | 优化了 XX |

检查方法：数每句话的动词数量。超过 2 个 → 必须精简。

---

## 准则 4：禁止模糊空话（零容忍）

模糊评价词不传递信息："存在明显差异"未说明差异内容，"具有重要价值"未说明价值所在。只有"将准确率提升至 60%"这类具体表述才有信息量。

### 绝对禁用词表（出现即改，无例外）

| 禁用 | 必须替换为 |
|------|-----------|
| 存在明显差异 / 差异明显 / 显著分异 | 写出具体差异内容："呈现单中心型、带状型、多核心型三种聚集模式" |
| 呈现明显的梯度特征 / 梯度明显 | 写出具体梯度："核心区占 XX%，边缘区占 YY%，分支区占 ZZ%" |
| 至关重要 / 举足轻重 / 非常重要 | 改为具体能力描述："能够将 R² 提升 0.11" |
| 非常明显 / 十分显著 / 效果显著 | 改为具体数字："R² 从 0.689 提升至 0.799" |
| 重要意义 / 重要价值 / 重大贡献 | 改为具体解决了什么问题 |
| 具有一定的参考价值 | 改为具体为什么提供了什么参考 |
| 理想 / 良好 / 优异 | 改为具体指标："R² = 0.948, RMSE = XX" |
| 关键任务 / 关键方法 / 关键抓手 | 改为具体动作："构建 XX 数据集""识别非线性阈值" |
| 贡献总增量 / 贡献总增长 | 改为明确占比："占增长总量的 64%" |
| 趋于平缓 / 趋于稳定（单独出现） | 必须给出对比基准："A 组在 2020 年后增速放缓（相对 2015-2019）" |
| 已有研究表明 / 研究表明（句首） | 直接写结论，删除虚引导："X 与 Y 正相关" |
| 在 XX 方面（段首/句首） | 让 XX 作主语 |
| 差异化策略 / 差异化分析 | 改为具体差异内容 |
| 采用多种方法 / 涵盖多个维度 / 大量研究 | 写出数量并列名："包括 A、B、C、D 四种方法"；列不出 → 这句是无用中心句，删 |

### "多种 / 多个 / 大量"是无用中心句

"采用多种方法分析 X"中的"多种"没有信息量：读者要么数得出几种、看得见哪几种，要么这句话等于没写。中心句想强调什么就直接写什么：想强调方法，列出方法名；想强调两条分析主线，直接写"X 的分析包括 [层面 A] 与 [层面 B] 两个层面"。

检查方法：全文搜索上述禁用词，逐个替换。一个都不能留。

## 准则 5：消灭"对"字

"对"不是真正的动词，只会让句子变模糊。

"对 X 的 Y"结构把动作藏在介词后面，读者无法立即识别谓语；改写为"Y X"或写出具体动词，动作即可外显。

| 反面 | 正面 |
|------|------|
| 对系统性能的优化 | 优化系统性能 |
| 对模型精度进行验证 | 验证模型精度 |
| 对实验结果的特征进行分析 | 分析实验结果的特征 |
| 对结果的可靠性对比 | 验证结果可靠性 |

检查方法：全文搜索"对"，逐个判断能否删除或替换为具体动词。

## 准则 6：句间逻辑关系

段落内每句话之间必须有逻辑关系，不能是无关句子的堆砌。

段内相邻两句之间必须存在可命名的逻辑关系。逐条罗列"某某发现 A、某某提出 B"而不交代彼此关系，是文献综述最常见的失败形态。

四种合法逻辑关系：
- **并列**："A 方面，……；B 方面，……"
- **递进**："首先获取 X，进而将其应用于 Y"
- **因果**："由于 A 精度最高，因此选择其作为最终模型"
- **转折**："已有研究集中于 A，但尚未涉及 B"

段落结构只有三种：总分、总分总、A 转折 B。

### "因此/所以"前后必须真因果

"因此/所以"的前后必须构成可推导的因果。前文与结论之间没有推导关系时，"因此"只是装饰性连接词，应删除或重写前文。

检查方法：
1. 全文搜索"因此 / 所以 / 故"
2. 遮住后一句，能否从前一句**推导**出后一句？
3. 推不出 → "因此"是连接剂不是真因果 → 删除或重写前句

示例：
| 反面 | 正面 |
|---|---|
| A 方法已发展多年，B 模型也被提出。**因此**获取最新 X 数据是精细化研究的基础。 | 主流方法有 A 和 B：A 的缺陷是 XX，B 的缺陷是 YY。**因此**需要构建更精细的核算数据集。 |

## 准则 7：总分结构

先告诉读者"一共几件事"，再逐一展开。

总分结构是最稳定的段落骨架："A 包括 B、C、D；B ……；C ……；D ……"。绝大多数规范段落均按此展开。

| 反面 | 正面 |
|------|------|
| 首先做了 X，然后做了 Y，接着 Z，最后还加入 W…… | 数据预处理包括三个阶段：A、B、C。A 阶段，……；B 阶段，……；C 阶段，……|

### 工作数量必须外显

多项工作合写在连续语流中，读者难以数出件数。应当用连词、编号或动词领起，使工作数量一眼可数。

让读者"数得出你做了几件事"的三种工具：
1. **连词开头**：首先 / 其次 / 最后（三件及以下）
2. **编号**：一是 / 二是 / 三是；或 ①②③
3. **不同动词开头**：构建了 X；提出了 Y；优化了 Z

检查方法：读完一段，能否一口气说出"这段说了几件事"？说不出 → 工作数量未外显 → 加连词/编号/动词领起。

## 准则 8：字字有用

可有可无的字必然是无。

判断标准：某字、某词在全文中可有可无时，删除后语义不受损失，即应删除。

必须删除的虚词：对于、其中、始终、进而（无递进时）、从而（无因果时）。

### 虚引导词清单（必删）

| 虚引导 | 出现位置 | 处理 |
|---|---|---|
| 已有研究表明 / 研究表明 | 句首 | 直接写结论："X 与 Y 正相关" |
| 已有研究 / 现有研究 / 现有学者（作主语） | 句首 | 删掉，直接以方法或对象作主语并挂文献："[方法 A] 已用于……" |
| 在 XX 方面 | 段首 / 句首 | 让 XX 作主语 |
| 将其作为 XX 进行 XX | 任意 | 拆成简单主谓宾 |
| 该方法已被应用于 | 任意 | "X 已应用于 A 和 B 两方面：……" |
| 基于 XX 方法（动宾前） | 动宾前 | 删除"基于"，直接用方法名作工具状语 |

### 同义反复必删

定义本身为真的话没有信息量，写出来等于考验读者的耐心。

| 反面（同义反复） | 问题 | 正面 |
|---|---|---|
| 总量随规模增大而增大 | 由定义即可知为真，没有信息量 | 写出真正想表达的对比："A 组中 X 占 90%，B 组中 X 仅占 10%" |
| 时间跨度覆盖了整个研究期 | 研究期就是时间跨度 | 数据覆盖 2015—2024 共 10 年 |

### 动词时态错位：表结果的词不用于介绍方法

"表明 / 揭示 / 发现"是陈述**结果**的词。介绍一个方法**能做什么**时，用"刻画 / 度量 / 识别 / 分析"。

| 反面 | 正面 |
|---|---|
| 该方法表明数据的分布特征 | 该方法刻画数据的分布特征 |
| 该指标揭示样本差异 | 该指标度量样本间的差异程度 |

## 准则 9：方法命名统一

同一篇论文中方法命名出现 3 种以上形态（纯英文缩写、纯中文、中文加缩写混用），会直接破坏专业感。命名风格必须全文统一。

### 统一规则（二选一，不能混用）

**方案 A：全中文方法类别**

把英文缩写转译成中文方法类别。例如：
- PCA → 主成分分析
- t-SNE / UMAP → 降维可视化
- SHAP → 特征归因
- CatBoost / XGBoost → 梯度提升树

**方案 B：全"中文（英文缩写）"**

首次出现：**中文全称（英文缩写）** 或 **中文全称（英文全称，英文缩写）**
- 主成分分析（Principal Component Analysis, PCA）
- 偏依赖图（Partial Dependence Plot, PDP）
- 沙普利可加性解释（SHapley Additive exPlanations, SHAP）
- 卷积神经网络（Convolutional Neural Network, CNN）

### 人名方法的处理

Mann-Kendall、Pearson、CatBoost、SHAP、Adam 等是**人名或工具名**，不是缩写——保留英文原拼即可，不强制配中文译名。对应的**中文部分落在方法类型描述上**：
- Mann-Kendall **趋势检验**
- Adam **优化器**
- Pearson **相关系数**

检查方法：通读全文，方法命名风格是否一致。出现 3 种以上命名形态 → 违反。

## 准则 10：禁自造词 / 禁张冠李戴

### 类别命名必须用标准词

空间形态的类别命名只能使用标准形状词（带状、团状、中心-外围等）。"港口集聚""双核驱动"不是形状描述，属于自造词。

合法的空间形态类别：**带状 / 团状 / 中心-外围 / 多中心 / 单中心 / 哑铃型 / 矩形型**

禁止自造：
- ❌ 双核驱动、港口集聚、三核轴向（不是形状）
- ❌ 自造缩写 + 错配"走廊"
- ❌ 把多个城市合称成一个"集聚区"

### 指标命名必须成立的中文搭配

指标名称必须是成立的中文搭配。词语组合不成立（如"处理量速度"），读者会直接判定为病句。

❌ "处理量速度"、"性能效果"、"鲁棒鲁棒性"（语法搭配病句）
✅ "处理速度"、"处理通量"、"性能"、"效果"、"鲁棒性"

## 准则 11：改进要落到具体指标

改进描述绕再多文字，不如一个具体数字。"将空间分辨率从 30 米提升到 10 米"一句即可让读者理解改进幅度。

### 禁止只说方法论标签

❌ 只说"本研究采用自下而上核算"——不够
✅ 必须紧跟具体指标提升：
- 空间分辨率：30m → 10m
- 行政单元：省级 → 市级 → 县级
- 覆盖类别：X 类 → Y 类
- R² / RMSE 的具体数字
- 覆盖期间：2015—2024（10 年面板）

### 批评要批到点子上

对前人工作的批评必须指向具体缺陷。既不说明前人做到了什么程度、也不指出差距所在的批评，没有论证效力。

| 反面（空批评） | 正面（落到点） |
|---|---|
| 现有方法精度不足 | 现有方法只能达到 [粗粒度]，无法识别 [细粒度] 差异 |
| 现有指标存在局限 | 现有指标无法揭示 [关键维度] 的内部结构（如 A-B-C 三层关系）|
| 以往研究较为粗略 | 以往研究多使用单一时点/单一来源，缺乏 N 年/N 来源面板 |

### 批评的三道检验（写"前人不足"前必过）

1. **因果成立测试**：说"前人没考虑 X"之前，必须先论证 X 与研究对象有关。逻辑顺序是"研究对象的 [某特征] 与 X 有关 → 前人未纳入 X → 难以刻画 [某特征]"。先有关、再说缺，否则读者会反问"X 凭什么影响研究对象"。
   - 反面：前人只使用 [指标 X]，未考虑 [指标 Y]，所以存在不足。（Y 与研究对象的关系未经论证，批评不成立）
   - 正面：研究对象的 [特征] 与 [指标 Y 所度量的性质] 有关；前人未纳入 [指标 Y]，难以刻画该特征。
2. **接回对象测试**：方法层面的批评（线性假设、未考虑某结构）必须接回研究对象——对象的哪个性质决定了这条缺陷在本研究中是问题。只批方法不提对象，批评悬空。
3. **可移植性测试**：把这段批评整体移植到另一篇不同主题的论文里，若照样成立，说明它没有写出任何与本研究对象有关的内容 → 重写。

### 引入新方法必须给出"亮点结论"

仅写"引入了 [新方法]"不构成论证。必须说明它解决了旧方法的什么问题，并给出用它得到、旧方法得不到的结论。

新方法三件套：
1. **一句话点破**："[新方法] 解决了 [旧方法] 不能处理 [某问题] 的缺陷"。
2. **亮点结论**：应用到本研究后，写出旧方法无法识别、新方法识别出的具体发现。
3. **对比小例**（最好有）：同一数据分别用新旧方法处理，结果差异即优势证明。

同时写明适用边界：指出新方法与旧方法结果趋同的退化条件，论证反而更可信。

## 准则 12：语义直白，朗读可过

单字均可识别而整句语义不明的句子，是最需要重写的句子。论文面向的是不熟悉该方向的读者，表达必须经得起逐句检验。

### 三个检验（输出前全过）

1. **朗读检验**：逐字默读全文（或出声读一遍）。出现卡顿、需要回读的位置，即是不通顺的位置。
2. **复述检验**：每个关键句应能用口语向非本方向的读者复述。无法复述说明概念尚未想清，应先理清再落笔。
3. **翻译检验**：设想句子译成英文后是否仍可理解。依赖语感堆叠的抽象词在翻译后会失去全部语义。

### 概念词自查

句中出现抽象概念词（"体系""机制""框架"等）时，自问：**能举出一个具体实例吗？** 举不出实例的概念词应替换为直白表述。

| 反面 | 问题 | 正面 |
|---|---|---|
| 该模型与任务需求之间存在深层的结构性适配 | "结构性适配"无实指，读者只能猜 | 该模型的感受野为 X，可覆盖平均尺寸为 Y 的目标 |
| 该框架体现了多层次的内在关联 | "多层次""内在关联"均无实指 | 该框架包含数据、特征、决策三层，相邻层以 [接口] 传递结果 |

### 专有名称与数字核对

机构、标准、数据产品等专有名称按官方原文逐字书写；统计数字与统计口径必须与数据来源一致，不同口径不得混用。

检查方法：通读全文一遍；对每个抽象概念词做举例自问；对每个专名与数字回到原始来源核对。
