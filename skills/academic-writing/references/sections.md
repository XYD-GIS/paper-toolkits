# 章节写作规范（引言 / 方法 / 结果 / 讨论 / 结论）

---

# 一、引言写作

## 引言必须构建完整因果链

每一步要有逻辑依据：
1. 全局背景（领域/问题严峻）
2. 战略目标 / 政策响应
3. 现有路径的局限
4. 替代路径的方向
5. 该方向需要先解决什么前置问题
6. 因此开展本研究

反面案例："X 很严峻 → 国家提出 Y → 所以我研究 Z"——每句话都对，但放在一起没有因果关系，是"废链"。

## 文献综述：以方法/主题为主线，不以人名罗列

文献综述不需要罗列作者与样区，只需要呈现方法、方法得出的主要结论与存在的主要缺陷。

反面：「Zhang (2019) 用 X 发现 A。Li (2020) 用 Y 发现 B。Wang (2021) 用 Z 发现 C。」

正面：
> 「该领域研究主要分为 X 识别与 Y 探测两类。X 方面，已有研究采用 [方法 A]、[方法 B] 等方法，揭示了宏观格局；Y 方面，[方法 C] 是主流方法，已识别出 [因素 1]、[因素 2]、[因素 3] 等因子的解释力差异。然而，跨方法整合与分类差异分析仍较少见。」

### 标准三段式写法

```
中心句（研究有几类指标 / 几种方法）
├─ 第一类：方法 → 代表结论 → 局限
├─ 第二类：方法 → 代表结论 → 局限
├─ 第三类：方法 → 代表结论 → 局限
└─ 转折 → 所有类均存在共同缺陷 → 引出本研究方法
```

### 转折落脚必须点到核心维度

批评某类方法的不足时，落脚点必须与本研究将要弥补的核心维度一致；落在无关维度上，转折无法引出本研究。

✅ 典型转折："以上方法均不能揭示 [对象] 内部的**结构关系**（核心-边缘-桥接 / 因果链 / 层级关系），而 [本研究采用的方法] 正可弥补这一缺陷。"

批评前人不足前，必须通过准则 11 的三道检验（因果成立 / 接回对象 / 可移植性），尤其是：先论证被忽略的维度与研究对象有关，再说前人没做。

### 综述先定主线：对象 / 内容 / 方法 / 结论不混写

四个概念先分清：**研究对象**（研究的是什么东西）、**研究内容**（对象的哪几个特征）、**研究方法**（用什么工具）、**研究结论**（得到什么模式）。一段综述只能选一条主线：

- 以**方法**为主线："X 的分析方法包括 A、B、C 三类。A 刻画……；B 度量……；C 识别……"
- 以**内容**为主线："X 的研究包括 [层面 A] 与 [层面 B] 两个层面。[层面 A] 上，……；[层面 B] 上，……"

最常见的失败：中心句以方法为主线，第二句却滑到研究对象（"多以 [某类对象] 为对象"）——主线中途切换，读者跟丢。

检查方法：写完把每句的中心词摘出来排一列，问自己"我写的是方法还是内容"；中心句说什么，全段就只说什么。

### 例子与方法作用必须对应

综述里"方法 → 作用 → 例证"三者要扣死：说某方法刻画 [特征 A] 与 [特征 B]，例证就必须同时包含 A 与 B；说某指标区分 [层级 1] 与 [层级 2]，例证就要写明各发现属于哪一层级。作用说了 A，例子只给 B，等于没有支撑。

### 正文用到的方法，综述必须全覆盖

正文使用的每个方法，综述里都要出现。综述若以"需综合运用上述方法"收尾而正文只使用其中一部分，读者会立即质疑取舍依据；反过来，压缩综述时不能删掉自己使用的方法，否则正文方法失去铺垫。

### 学科 / 方向分工：写"强调什么"，不写"借助什么手段"

研究手段为各学科共享（文献分析、实地调查均非某一学科专属），按"A 学科借助 [手段] 研究 X"组织学科综述容易被反例推翻。学科综述应按**各学科强调的维度**分工：

> "X 的研究主要分属 A、B、C 三个学科：A 学科强调 [维度 1]，B 学科强调 [维度 2]，C 学科强调 [维度 3]。"

非本研究主线的学科写两三行即可，主线学科展开写——篇幅跟随重要性。

### 跨学科必要性从研究对象推出，不从"谁都做过"推

"各学科分别开展过研究"不构成跨学科的理由。必要性只能从**研究对象自身的完整流程**推出：

> "X 的 [总体目标] 包含 [环节 1]、[环节 2]、[环节 3]，三个环节分属不同学科，因此需要跨学科研究。"

注意时态：若写"多学科融合**已成为**重要方向"，那它是现状不是缺口，不能再当作本研究的创新空间。

### 指标体系综述按维度演进写

指标体系类综述的主线是**维度**（[维度 A] → [维度 B] → ……），写"先有哪个维度、谁加入了哪个维度、当前共几个维度、还缺哪个维度"。具体指标名是维度下的二级内容，不作主语——二级内容当主语，层级结构就乱了。

> "指标体系先以 [维度 A] 为主；随后 [维度 B] 被纳入；当前研究多涵盖 A、B、C 三个维度。然而，[维度 D] 尚未纳入，而 [对象] 的 [特征] 与 [维度 D] 有关。"

### 篇幅与重要性匹配

段落显得过长，本质是内容的重要性撑不起篇幅。与本研究主线无关的内容一句话概括（"以往方法主要有 A—G，用于 ……"）；与主线直接相关的内容分门别类逐句展开。压缩或扩写之前，先判断该内容对论证的重要程度。

### 文献随写随核

综述里每句带文献的话，写的时候打开原文核对——作者、年份、结论是否真的如此。写作中途读到特别合适的文献，立即存档并加入，不要等"最后统一加引用"：最后补挂的文献最容易出现张冠李戴和对不上号。

# 二、方法写作

方法只写三样：**原理 + 公式 + 变量解释**。

方法部分应保持纯净：只写方法本身，不混入结果。

检验标准：如果研究区/样本从 A 改为 B，这段方法描述是否仍然成立？不成立 → 混入了实验数据 → 删除。

## 算法/模型描述模板

1. 一句话说核心思想
2. 公式
3. "式中，X 代表……，Y 代表……" 逐一解释
4. （可选）结果区间："q 取值范围 [0, 1]，q 越大表示解释力越强"

## 模型应用描述模板

> "CatBoost 模型主要应用于研究的两个阶段。第一阶段用于学习 X 与多元特征的映射关系；第二阶段用于评估 Y 指标的重要性。"

## 数据构建 / 方法增强描述

必须回答三个问题：
1. **前人怎么做**：主流方法的精度/粒度/覆盖范围？
2. **问题在哪**：缺陷是"分辨率不够"、"只覆盖 X 子类别"还是"忽略了 Y 维度"？
3. **怎么改进**：不是"我们的方法更优"标签，而是给具体指标提升（分辨率 30→10、覆盖 X→Y、R² 0.85→0.94）

方法论标签（自上而下、自下而上、端到端）本身不传递改进信息。读者关心的是具体好在哪：精度提升、分析单元更细，还是覆盖范围更广。

### 错误写法 → 正确写法

| 错误（只贴标签） | 正确（落到指标） |
|---|---|
| 传统方法自上而下，我们自下而上 | 主流方法在 [粗粒度] 上达到精度 X；本研究在 [细粒度] 上达到精度 Y（提升 Z%）|
| 通过 [方法] 提高分辨率 | 通过 [方法] 将 [数据维度] 从 30m × 30m 聚合至 1m × 1m 单元 |
| 方法新颖、精度更高 | 关键指标提升：R² 从 0.85 → 0.94；RMSE 从 X → Y |

### 标签 → 人话翻译

❌ "通过 [方法] 实现了端到端建模"
✅ "现有数据主要在 [粗粒度] 发布（较为宏观），本研究通过 [方法] 将其映射至 [细粒度]。"

## 引入新方法：必须写出亮点结论

写"为弥补上述缺陷，引入 [新方法]"之后，必须跟上它在本研究中得到、旧方法得不到的具体发现；只换方法名不给增量结论，读者会怀疑是把简单问题复杂化。完整写法见准则 11"新方法三件套"（一句话点破 + 亮点结论 + 对比小例 + 退化条件）。

# 三、结果写作

先给结论，再给数据。大量数字放表格，正文只写极值、拐点与模式命名。

## 时空分析分段呈现

- "时间维度上，X 呈三阶段演化特征：起源与发展阶段（先秦—汉晋）共 a 项，繁荣阶段（隋唐—宋元）增至 b 项，成熟阶段（明清—近代）达 c 项。"
- "空间维度上，X 呈现单中心型、带状型、多核心型三种模式。"

**绝不写"差异明显""分布不均匀"**，必须命名具体模式。

## 时间规律写法陷阱

必须拆成独立断言，一句讲一个维度：
```
总量变化：X 由 a 增至 b（2015 → 2024）
增速排名：增长最快的是 X，最慢的是 Y
拐点识别：A 拐点出现在 N 年；B 拐点出现在 M 年
占比演变：A 贡献增长总量的 64%
```

## 空间规律：必须用形状词分类

合法分类：带状 / 团状 / 中心-外围 / 哑铃型 / 多中心 / 单中心
不合法：双核驱动、港口集聚、X-Y-Z 制造业走廊、三大核心区

## 影响因素：先整体，后分群，最后落管理含义

呈现顺序：整体主导因子 → 各分组的差异 → 可调控的变量。

**管理含义落点模板**：
> 首要因子虽为 X（某静态/结构性指标），但该指标刚性较强、难以短期调整；真正可操作的抓手是 Y 类细部指标（结构比率、配置参数、流程参数等），它们对应可执行的调整动作。

# 四、讨论写作

按**模式/类别分类讨论**，绝不按地区/分组逐个重复。

| 反面 | 正面 |
|---|---|
| A 组呈 X 模式，原因…；B 组呈 Y 模式，原因…；C 组呈 Z 模式，原因…（5 组写 5 段） | 第 1 段：X 模式的形成机制（共同因子）；第 2 段：Y 模式的形成机制；第 3 段：影响因子层级（自然底色 + 人文驱动）；第 4 段：与已有研究对比；第 5 段：政策/应用含义 |

讨论要点：
- 交互效应必须解释政策含义："同时调整 X 和 Y，效果大于分别调整"
- 政策建议专设讨论子节，不放结论
- 用人话描述模型结果，不用"形态学比值在特定转变处集中"这种表述

## 无独立结论章时：讨论按两大块写

部分期刊讨论与结论合并。此时讨论只写两大块，不按研究内容逐段铺：

1. **与同类研究比较的优势**：选 2—3 个方面（如数据覆盖、分析单元、方法），每个方面写"同类研究怎么做的（挂 2—3 篇文献）→ 本研究怎么做 → 好在哪"。比较必须有具体对象，不能只说自己好。
2. **主要结论 + 展望**：结论要定量（落到数字与模式名）；展望与不足合计两三句即可。

# 五、结论写作

短论文结论要短：只概括"做了 X，发现了 Y，揭示了 Z"。不列详细数字（结果部分已有）。结论中每一条都必须在结果/讨论中有对应分析。

模板：
> 「本研究以 X 为研究对象，整合 A、B、C 方法，揭示了 Y 的规律与机制。结果表明：(1) 时间上呈 ……；(2) 空间/分类上呈 ……；(3) 主导因子为 ……，交互机制以 …… 为主。本研究为 …… 提供了 …… 依据。」

## 创新点 / 贡献的写法

- **最多三条**。超过三条说明没归并：能合并的合并（同属数据层的两条算一条）。
- **写具体动作 + 数字**，不写抽象的范围话或方法论标签。
  - ❌ "将研究范围由局部扩展至整体"（只有口号，没有具体动作与结果）
  - ✅ "构建了覆盖 [N] 个样本的 [属性] 数据集，并识别出 [M] 类典型模式"
- 每条贡献都要能在结果章找到对应小节；找不到 → 删。
